## 原则（三行）

1. **先 Issue，再写代码**；一个 PR 只关一个 Issue。  
2. **改动范围尽量小**；大需求拆成多个 Issue。  
3. **仓库保持干净**；不提交多余文件、密钥、本地工具目录。
